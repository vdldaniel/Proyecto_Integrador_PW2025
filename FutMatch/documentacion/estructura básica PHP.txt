CÓMO LLAMAR LINKS:
-----------------------------------------------------------------------------
Fijarse en el archivo config.php cómo están declarados los distintos links
del proyecto. Es importante tener en cuenta esto si mueven archivos de lugar.

Si llegan a cambiar el nombre de un archivo o el lugar de un archivo, tienen
que actualizar correspondientemente config.php. Una vez hecho esto, no hace 
falta modificar ningun otro archivo del proyecto que llame a ese link ya 
que está estandarizado para minimizar el riesgo de conflicto. 

IMPORTANTE: 
- head.php ahora incluye <!DOCTYPE html> y <html lang="es" data-bs-theme="dark">
- Todas las páginas usan require_once con las constantes de config.php
- Los CSS específicos de página van en src/styles/pages/
- Los JS específicos de página van en src/scripts/pages/
- La variable $current_page activa automáticamente el botón de la navbar
- El tema oscuro (dark mode) se aplica automáticamente en todas las páginas


-----------------------------------------------------------------------------
ESTANDARIZACIÓN DE ESTILOS:
-----------------------------------------------------------------------------
Se estandarizaron los estilos de CSS para unificarlos:
    * base.css = contiene variables globales y tipografías
    * layout = estilos de navbar, grids, sidebar, containers, modales
    * components.css = botones, cards, badges, forms
    * pages\ = contiene los estilos específicos de las páginas
               (solo cuando son muy complejas)
Si se agrega un archivo de estilo no olvidar agregar a config.php
y luego llamarlo con el nombre de variable asignado
